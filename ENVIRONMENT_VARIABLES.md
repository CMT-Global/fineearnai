# Environment Variables Documentation

## Overview
This document tracks all environment variables used in the FineEarn platform, including those that are currently unused but may be referenced in legacy code or documentation.

---

## Active Environment Variables

### Supabase Configuration
- `VITE_SUPABASE_URL` - Supabase project URL (required)
- `VITE_SUPABASE_PUBLISHABLE_KEY` - Supabase anon/public key (required)
- `VITE_SUPABASE_PROJECT_ID` - Supabase project identifier (required)
- `SUPABASE_URL` - Backend Supabase URL (Edge Functions)
- `SUPABASE_SERVICE_ROLE_KEY` - Backend service role key (Edge Functions)

### Payment Processors - Active
- `CPAY_API_PUBLIC_KEY` - CPAY public key for deposits
- `CPAY_API_PRIVATE_KEY` - CPAY private key for API signatures
- `CPAY_ACCOUNT_ID` - CPAY merchant account identifier

### Email Services
- `RESEND_API_KEY` - Resend email service API key

### Currency Exchange
- `OPENEXCHANGERATES_APP_ID` - Open Exchange Rates API for currency conversion

### IP Geolocation
- `IPSTACK_API_KEY` - IPStack API for user location tracking

---

## ⚠️ Unused/Deprecated Environment Variables

### Payeer (NOT INTEGRATED - DO NOT USE)
The following Payeer environment variables are **NOT used** in the current system. Payeer integration was never completed and all related code has been removed as of Phase 2 cleanup.

- ❌ `PAYEER_API_ID` - **UNUSED** - Payeer merchant API ID
- ❌ `PAYEER_API_KEY` - **UNUSED** - Payeer API secret key  
- ❌ `PAYEER_ACCOUNT` - **UNUSED** - Payeer account number

**Important Notes:**
- **Payeer is NOT integrated** into the platform
- The platform previously had 200+ lines of unused Payeer code that has been removed
- Any references to Payeer in legacy documentation should be ignored
- Withdrawals are processed through **CPAY or manual processing only**

**Why Payeer Variables Exist:**
These variables may still be referenced in:
- Legacy documentation
- Old migration files
- Historical code comments
- Environment variable templates

**Action Required:**
- Do NOT set these variables
- Do NOT attempt to use Payeer for withdrawals
- If you see code referencing Payeer, it's obsolete and should be removed

## Current Withdrawal Processing Method

As of **Phase 2 Implementation (2025-10-23):**

### Deposits
✅ **CPAY** - Fully integrated for cryptocurrency deposits

### Withdrawals  
✅ **Manual Processing** - Admin manually sends crypto from external wallet
- Admin reviews withdrawal request
- Admin approves request (status: `approved_manual`)
- Admin sends crypto from external wallet
- Admin enters blockchain transaction hash
- System marks withdrawal as completed

**No automated withdrawal API is currently integrated.**

## Adding New Environment Variables

When adding new environment variables:
1. Add them to `.env` file (auto-generated by Supabase)
2. Document them in this file under "Active Environment Variables"
3. Update the relevant Edge Functions that use them
4. Never commit actual secret values to version control
5. Use Supabase's secrets management for sensitive values

---

## Security Best Practices

- ✅ All sensitive keys stored in Supabase secrets (not in `.env`)
- ✅ Never expose private keys in client-side code
- ✅ Use `VITE_` prefix only for public/safe client-side variables
- ✅ Backend-only variables (Edge Functions) should NOT have `VITE_` prefix
- ✅ Rotate API keys regularly
- ✅ Monitor API usage for anomalies

## Troubleshooting

### "Payeer API error" or "Payeer not configured"
**Solution:** Ignore these errors. Payeer is not integrated. The system now uses manual processing for withdrawals.

### "Cannot find PAYEER_* environment variable"
**Solution:** This is expected. These variables are not used. If code is checking for them, that code is obsolete and should be removed.

### "Withdrawal failed to process"
**Solution:** Check that:
1. `cpay-withdraw` Edge Function is deployed
2. Admin has approved the withdrawal
3. Withdrawal status is tracked correctly in database
